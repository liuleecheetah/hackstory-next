# HackStory

**多人協作、模組化的視覺化時間軸工具。**

面對龐雜的公共議題，討論常常卡在「連基本事實都各說各話」。現有工具（TimelineJS、時間地圖）只呈現一顆顆孤立的事件，看不出事件之間的因果，也無法把不同人整理的時間軸放在一起對比。HackStory 想補上這一塊——讓一般公民、倡議團體、新媒體，都能把龐雜的資訊整理成看得懂、能對照、可分享的時間歷程。

源自 2016–2017 年的 [g0v](https://g0v.tw) 專案（提案人：劉李俊達、吳翊瑋、黃嘉旭），當年因缺乏前端工程師而停滯，現以 Claude Code 重啟。

---

## 三個差異化定位（專案的靈魂）

1. **多軸對比** — 同時呈現多條時間軸，看出彼此的互動與呼應。
2. **可組合** — 一條時間軸是一個獨立的 `.hst.json` 檔案，能下載、備份、寄給別人、與他人的疊加混合。
3. **相對時間** — 不知道確切日期、只知道先後順序，也能把事件放上軸（「在 A 之後、B 之前」）。

---

## 現在能做什麼

**看與比對**
- 多條時間軸上下並排，各有顏色；同時載入多份檔案疊加對比
- 滑鼠滾輪縮放、拖曳平移，日／週／月／年尺度切換
- 大段沒有事件的空白自動摺疊（斷軸記號標示「略過 N 年」）
- 點事件看詳情卡：完整說明、地點、標籤、可點的來源連結、查證程度
- 事件之間的**關係線**（導致／回應／衍生／矛盾／同一事件），滑過或點選就亮起
- **相對時間**事件以虛線推估位置呈現，明確標示「不是真實日期」
- 進行中的事件畫到「今天」並在右端淡出

**編輯（全程不需要寫 JSON）**
- 雙擊空白處新增事件、點事件直接編輯、刪除（連帶清理關係線與相對時間參考，保證不留壞資料）
- 畫面上建立／刪除關係線、把事件設成相對時間、標示關鍵事件（放大顯示）
- 新增／改名／刪除軸線，「＋空白時間軸」從一張白紙開始
- 所有修改自動存成瀏覽器草稿、多步復原／重做、關頁前提醒

**匯入與分享（不需要後端）**
- 上傳 CSV、貼公開的 Google 試算表網址；匯入預覽會攤開「成功／警告／無法解析」讓你逐筆修正（**永不靜默丟資料**）
- 匯出 `.hst.json`、SVG／PNG 圖片、iframe 嵌入碼
- 分享連結（`?src=公開網址`）：把檔案放上 GitHub／Gist 或用公開試算表，連結一貼別人就能看；分享頁預設唯讀，可「建立可編輯副本」

詳細操作見 [docs/import-guide.md](docs/import-guide.md)。

## 快速開始

```bash
npm install
npm run dev        # 開發伺服器，預設 http://localhost:5173
```

其他指令：

```bash
npm test           # 執行 Vitest 測試
npm run build      # TypeScript 型別檢查 + 正式版建置
npm run preview    # 預覽正式版建置結果
```

推送到 `main` 分支會經 GitHub Actions 自動測試、建置、部署到 GitHub Pages。

## 架構：六層，嚴格分離

```
6. ui/        頁面外殼、工具列、面板、對話框
5. share/     發佈／取用共用庫              ← Phase 2，位置預留
4. compose/   圖層管理（疊加多份文件、配色、顯示隱藏、復原重做）
3. render/    視覺化引擎（SVG 手繪、時間尺度、空白摺疊、關係線、相對時間求解呈現）
2. adapters/  匯入匯出（CSV / Google Sheet / JSON / 圖片 / 分享連結）
1. core/      資料模型、SPEC 驗證、時間解析、相對時間求解
```

**鐵律：每一層只能呼叫比它低的層。** `core` 不 import 任何其他層、不碰畫面；`render` 不知道資料從哪來。新增匯入格式只動 `adapters/`，換畫法只動 `render/`，接資料庫只動 `share/`。`core/` 層有完整的 Vitest 測試。

## 技術棧

| 項目 | 選擇 |
|---|---|
| 建置 | Vite |
| 框架 | React + TypeScript |
| 樣式 | Tailwind CSS |
| 時間刻度計算 | `d3-scale` + `d3-time`（只借用計算，繪製自己用 SVG） |
| CSV 解析 | `papaparse` |
| 測試 | Vitest |
| 部署 | GitHub Pages |

繪製層刻意**不使用任何現成時間軸函式庫**（如 vis.js）：相對時間、關係線、空白摺疊、事件面積等比例等需求都違反那些函式庫的核心假設，繞過限制比自己畫還累。

## 檔案格式

一條時間軸就是一個 `.hst.json` 檔案。完整規格見 [SPEC.md](SPEC.md)；想直接動手的話，[examples/template.hst.json](examples/template.hst.json) 和 [examples/template.csv](examples/template.csv) 是可以拿去改的起手範本。

`examples/` 也收錄真實資料：台灣婚姻平權進程、食安事件、以及預設展示的「科幻預言與現實對照」（科幻小說寫下未來 vs 現實世界實現，用關係線串起「未來遲到的年數」）。`dirty-sample.csv` 故意保留各種壞資料，用來檢驗匯入器。
