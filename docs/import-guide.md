# HackStory 時間軸檔案編寫與匯入說明

這份說明寫給要把自己的資料做成時間軸的人。你不需要懂程式——會用試算表就能做出完整的時間軸；想要「關係線」等進階功能，再照第五節手寫 `.hst.json` 檔案即可。

**兩份可以直接拿去改的範本：**

| 範本 | 適合 | 位置 |
|---|---|---|
| 試算表範本 | 大多數人，從整理資料開始 | [examples/template.csv](../examples/template.csv) |
| `.hst.json` 範本 | 想用關係線、想手寫檔案的人 | [examples/template.hst.json](../examples/template.hst.json) |

---

## 一、三種把資料放進 HackStory 的方式

| 方式 | 適合的情況 | 在哪裡操作 |
|---|---|---|
| 上傳 CSV 檔案 | 資料在 Excel 或電腦裡 | 上方「匯入 CSV / Google Sheet」按鈕 |
| 貼上 Google 試算表網址 | 資料在 Google Sheet、想多人共同編輯 | 同上 |
| 載入 `.hst.json` 檔案 | 別人給你的時間軸檔，或你手寫的檔案 | 左側圖層面板「＋載入 .hst.json」 |

前兩種會先進入「**匯入預覽**」：看到哪些資料成功、哪些有問題，壞資料可以當場修正，確認後才變成圖層。

## 二、用試算表準備資料

**建議直接複製 [template.csv](../examples/template.csv) 開始改。** 規則如下：

- **第一列可以是主題標題**（例如「同婚大事記」）——匯入時會自動變成圖層標題；也可以省略，直接從表頭開始
- 表頭之後，每一列是一則事件
- 只有**日期**和**標題**必填，其他都可省略

### 欄位對照表（表頭中英文都認得，大小寫不拘）

| 欄位 | 接受的表頭寫法 | 必填 | 說明 |
|---|---|---|---|
| 日期 | `日期`、`開始日期`、`Date`、`Start Date` | ✅ | 寫法見下一節 |
| 標題 | `標題`、`事件`、`Title` | ✅ | 一句話講完，這是軸上直接顯示的文字 |
| 時間 | `時間`、`Time`、`Start Time` | | `14:00`，或時段 `09:00-18:00`（自動變成當天的區間） |
| 結束日期 | `結束日期`、`End Date` | | 填日期就畫成「一段期間」的長條；**寫「至今」**（或 迄今／持續中）表示事件還在進行，長條會一路畫到今天、右端淡出 |
| 說明 | `說明`、`描述`、`Description` | | 完整內容，**點開事件就會看到** |
| 地點 | `地點`、`Location` | | 例如 `立法院前` |
| 分類 | `分類`、`軸線`、`Track` | | 相同分類排在同一條軸線（見第四節） |
| 標籤 | `標籤`、`Tags` | | 逗號分隔，例如 `釋憲, 大法官` |
| 來源 | `來源`、`Source`、`URL` | | 貼網址最好——詳情卡上會變成可點的連結 |

認不得的表頭會被忽略，匯入預覽會列出「認得的欄位」讓你核對。

### 日期怎麼寫

分隔符號用斜線、橫線、點、國字都行，補零不補零都行：

| 你寫的 | HackStory 的理解 |
|---|---|
| `2017/5/24`、`2017-05-24`、`2017.5.24`、`2017年5月24日` | 精確到「日」 |
| `2010年6月`、`2010/6`、`2010-06` | 精確到「月」 |
| `1986`、`1986年` | 精確到「年」 |

**重要觀念：不知道確切日期，就不要假裝知道。** 只知道年月就只寫年月，畫面上會誠實顯示「2010/6」而不是隨便補一天。（畫面右上有「顯示事件日期」「含年份」選項，同一年內的時間軸可以把年份關掉，比較清爽。）

**會被攔下來的寫法**：`2017/2/30`（不存在的日期）、`2017/13/1`、`25:00`、`你好天`——全部進「待修正」清單讓你處理，不會混進時間軸，也不會被偷偷丟掉。

### Google 試算表要先設成公開

HackStory 不要求登入 Google、不碰你的帳號權限，所以試算表必須公開可讀，兩種設法擇一：

1. **發布到網路**（推薦）：檔案 → 共用 → 發布到網路 → 按「發布」，複製連結
2. **連結共用**：右上「共用」→「知道連結的使用者」（檢視者即可），複製網址列的網址

貼進匯入視窗按「讀取」。**圖層標題會自動帶入你的試算表名稱**，不滿意可以當場改，之後也能在面板按 ✎ 重新命名。一次匯入一個分頁；想匯入特定分頁，先在瀏覽器切到該分頁再複製網址。

### 遇到不完整的資料會怎樣（絕不偷偷丟資料）

| 你的資料長這樣 | HackStory 的處理 |
|---|---|
| 整列空白 | 自動略過 |
| 有日期、沒標題 | 略過，記在「警告」清單提醒你 |
| 日期看不懂、欄位錯位 | 進紅色「**待修正**」區：欄位直接改、按「重試」；不要就按「忽略」 |
| 同日期同標題重複出現 | 全部保留、標黃色「疑似重複」，由你決定要不要按 ✕ 移除 |

## 三、匯入之後，畫面上還能做這些事

- **點事件** → 詳情卡：完整標題、日期、說明、地點、標籤、可點的來源連結、查證程度
- 詳情卡按「**✎ 編輯事件**」→ 直接修改標題、日期、說明、地點、標籤、查證程度，或刪除事件（嵌入模式為唯讀，看不到編輯按鈕）
- **在軸線空白處點兩下 → 新增事件**：日期自動帶入你點的位置（可改），填上標題按「儲存」就出現在軸上
- **從一張白紙開始**：面板「＋空白時間軸」建立全新圖層 → ✎ 改名 → 雙擊加事件。每個圖層的軸線都能 ✎ 改名、「＋新增軸線」、刪除空軸線（有事件的軸線與最後一條會擋住不給刪）
- 詳情卡裡勾「**標示為關鍵事件**」→ 該事件在軸上放大、加光暈、粗體——用來突顯轉捩點
- 圖層面板：✎ 重新命名圖層、勾選顯示隱藏、▲▼ 排序、✕ 移除；**多軸線的檔案可以個別調整每條軸線的顏色**
- 右上選項：顯示事件日期／含年份／顯示關係線／**摺疊空白**（把整條軸上沒有事件的大段空白壓縮，軸上以 ⫽ 記號標示「略過 N 年」；檔案裡的 `display.collapseGaps` 可設定預設值）
- 「匯出／分享」：你在畫面上做的修改（改名、改色、關鍵事件）**下載 `.hst.json` 時都會保存**；也能匯出 SVG／PNG 圖片、複製 iframe 嵌入碼

### 分享連結：把時間軸變成一個網址（不用架站）

1. 「匯出／分享」下載 `.hst.json`，把檔案放上任何公開網址——最簡單的做法是 GitHub 或 Gist（用它的 raw 網址）
2. 回到「匯出／分享」的「**分享連結**」區，貼上那個網址 → 複製產生的連結，傳給任何人
3. 對方打開連結，看到的就是你分享的時間軸（乾淨的，不含其他圖層）；同區也會產生對應的 iframe 嵌入碼

**進階：** 直接貼「公開的 Google 試算表」網址當來源——對方每次打開連結，看到的都是試算表當下的最新內容，適合持續共筆的資料。想同時分享多份，在連結後面接 `&src=另一個網址`，開啟時會疊成多個圖層。

## 四、分類欄：一份表格畫成好幾條軸線

表格有「分類」欄時，相同分類的事件排在同一條軸線，例如「主線」一條、「支線」一條，上下並排對比。沒有分類欄就全部放進同一條軸線。

## 五、`.hst.json`：想要「關係線」就用這個（進階）

試算表能做的到第四節為止。**事件之間的關係線**（例如「A 導致 B」「相隔 104 年後成真」）目前只能寫在 `.hst.json` 檔案裡。

**做法：複製 [template.hst.json](../examples/template.hst.json) 改內容**，用任何文字編輯器（記事本也行）打開。結構長這樣：

```
{
  "hackstory": "0.1",          ← 固定寫這個
  "id": "my-first-timeline",   ← 英數與連字號，當作檔案的識別碼
  "meta": { ... },             ← 標題、作者、授權
  "tracks": [ ... ],           ← 軸線清單（可多條，各有顏色）
  "events": [ ... ],           ← 事件清單
  "relations": [ ... ],        ← 事件之間的關係（畫成關係線）
  "display": { ... }           ← 建議的顯示範圍
}
```

### 事件（events）常用欄位

| 欄位 | 必填 | 說明 |
|---|---|---|
| `id` | ✅ | 檔案內唯一，例如 `"evt-001"`。關係線靠它指認事件 |
| `track` | ✅ | 對應 `tracks` 裡某條軸線的 `id` |
| `title` | ✅ | 軸上顯示的一句話 |
| `start` | ✅ | `{ "value": "2024-01-15", "precision": "day" }`；precision 可為 `year`／`month`／`day`／`minute`，value 格式依序為 `"2024"`／`"2024-08"`／`"2024-01-15"`／`"2024-01-15T14:00"` |
| `end` | | 填了就畫成長條 |
| `ongoing` | | 填 `true` 表示事件**仍在持續**：長條畫到「今天」、右端淡出，詳情卡顯示「至今仍持續」。與 `end` 擇一使用 |
| `description` | | 點開事件看到的完整說明 |
| `importance` | | **填 `5` 就是關鍵事件**，在軸上放大顯示 |
| `confidence` | | `verified`（已查證）／`reported`（據報導）／`disputed`（有爭議）／`unknown` |
| `tags`、`location`、`sources` | | 標籤陣列、`{ "name": "地點" }`、`[{ "title": "...", "url": "..." }]` |

### 相對時間：只知道先後、不知道日期

`start` 可以不寫日期，改寫先後關係（`after`／`before` 填其他事件的 `id`，至少填一個）：

```json
"start": { "relative": { "after": "evt-001", "before": "evt-002" } }
```

畫面會在兩個參考事件之間畫一個**虛線空心圓點**，日期欄標示「（推估）」——明確告訴讀者這是推估位置、不是真實日期。點開詳情卡會寫出「在Ａ之後、在Ｂ之前」。相對事件可以指向另一個相對事件，串成先後鏈；互相循環或條件矛盾的事件不會亂畫，軸線標題會註記「N 筆相對時間無法推估」。

> **不想寫 JSON？畫面上就能設定**：編輯（或新增）事件時勾選「**改用相對時間**」，用下拉選單挑「之後／之前」的參考事件即可。

> **不想寫 JSON？可以直接在畫面上建立關係**：點事件 → 詳情卡「關係」區按「＋連到另一個事件」→ 點選目標事件 → 選類型、填說明 → 建立。詳情卡也能逐條刪除關係。以下格式說明給想手寫檔案的人。

### 關係（relations）——關係線的來源

```json
{ "from": "evt-002", "to": "evt-001", "type": "derives_from", "label": "顯示在線上的說明文字" }
```

| type | 意義 |
|---|---|
| `causes` | 導致 |
| `responds_to` | 回應／反制 |
| `derives_from` | 衍生自 |
| `contradicts` | 與之矛盾 |
| `same_event` | 同一事件的不同記載（畫成虛線） |

畫面上：關係線平常是淡灰色細線；**滑鼠移到事件上或點選事件**，相關的線會亮起並顯示 `label` 的文字。右上「顯示關係線」可整批開關。注意 `from`／`to` 只能指向**同一份檔案內**的事件——想連結兩組資料，把它們放進同一份檔案的兩條軸線（參考預設範例「科幻預言與現實對照」的檔案 [scifi-vs-reality.hst.json](../examples/scifi-vs-reality.hst.json)）。

### 檢查有沒有寫對

存檔後用「＋載入 .hst.json」讀進 HackStory——寫錯的地方會直接列出中文錯誤訊息（缺了什麼欄位、日期格式哪裡不對），照著改就好。完整技術規格在 [SPEC.md](../SPEC.md)。

## 六、常見問題

**問：說明、地點、標籤匯入後去哪了？**
答：點一下時間軸上的事件就會浮出詳情卡，全部都在那裡，也都完整保存在檔案裡。

**問：CSV 能不能寫關係線或關鍵事件？**
答：關係線不行（請用 `.hst.json`）；關鍵事件可以匯入後在詳情卡勾選，下載 `.hst.json` 時會保存。

**問：匯入後發現打錯字怎麼辦？**
答：**點事件 → 按「✎ 編輯事件」直接改**：標題、日期（吃所有匯入器支援的寫法，結束日期寫「至今」＝進行中）、說明、地點、標籤、查證程度都能改，也可以刪除事件（指向它的關係線會一併清掉）。改完用「匯出／分享」下載保存。圖層名稱、軸線顏色、關鍵事件同樣直接在畫面上改。

**問：可以匯入幾筆資料？**
答：幾百筆沒問題。太密的事件會自動往下疊排，放大就看清楚了。
